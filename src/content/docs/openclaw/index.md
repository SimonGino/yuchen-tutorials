## 本课程目标

读完整门课程，你将拥有一个可长期运行的个人 AI 助手系统：

- 安装部署并稳定运行

- 用 Skills 扩展能力

## OpenClaw 是什么

一句话总结：**OpenClaw 是一个可常驻运行的个人 AI 助手框架。**

它不是一个聊天网页，而是一套自托管的 AI 中枢：

- **Gateway** 跑在你自己的设备上（电脑、服务器、Mac Mini）
- **聊天渠道**（Telegram / 飞书 / 微信 / Discord）是你和助手的交互入口
- **模型**（云端 API / 本地模型）提供 AI 能力
- **Skills** 扩展助手的执行能力

```
你（聊天渠道）→ Gateway（你的设备）→ 模型（云端/本地）
                    ↕
              Skills / 工具 / 浏览器
```

和普通 AI 聊天工具的核心区别：

| | 普通 AI 聊天 | OpenClaw |
|---|---|---|
| 运行位置 | 别人的服务器 | 你自己的设备 |
| 数据控制 | 平台持有 | 你持有 |
| 执行能力 | 只能对话 | 可读写文件、跑命令、操控浏览器 |
| 持续性 | 关掉页面就没了 | 7x24 常驻运行 |
| 记忆 | 上下文窗口内 | 跨会话持久记忆 |

---

## 它能做什么

**日常助理**：整理待办、总结邮件、管理日程、语音转结构化清单

**信息收集**：定时扫描你关注的信息源，筛选后推送到 IM

**文件处理**：读文档出摘要、归档文件、生成周报

**浏览器自动化**：抓取网页、监控页面变化、操作 Web 应用

**开发辅助**：代码审查、日志分析、运维排障

**多 Agent 协作**：主助手分派子任务并行处理，汇总结果

---

## 你需要付出什么

- **一次性成本**：首次安装和配置，约 30-60 分钟
- **设备**：一台尽量在线的设备（当前电脑体验、云服务器或闲置设备长期用）
- **模型费用**：云端 API 按量计费，本地模型用算力换成本
