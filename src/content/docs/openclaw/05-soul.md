## 本节目标

- 写好 SOUL.md / IDENTITY.md / USER.md 三件套
- 助手从"通用 AI"变成"你的 AI"
- 重启后能通过 A/B 对比看到风格变化

---

## 1. 为什么要配置身份

安装完成后，助手用最通用的口吻回复你——像在和陌生人说话。

三件套的作用：

| 文件 | 作用 | 一句话说明 |
|------|------|-----------|
| `SOUL.md` | 定义性格 | 助手怎么说话、怎么做事、哪些不能做 |
| `IDENTITY.md` | 定义自我认知 | 助手的名字、形象、调性 |
| `USER.md` | 定义对你的了解 | 你的偏好、项目、工作方式 |

![身份三件套](https://oss.aiqqyc.com/2026/02/c43f8418a9629c7edc08cd97ae46e579.png)

这三个文件在 OpenClaw workspace 目录下。改了它们，助手的回复风格会立刻不一样。

---

## 2. SOUL.md：定义性格与边界

SOUL.md 是助手的"灵魂"。关键不是写形容词，而是写**行为规则**。

推荐结构：

```markdown
# SOUL

## Tone
- 直接、务实、不绕弯
- 不说"好问题！"和"很高兴能帮到你！"

## Style
- 先给结论，再给步骤
- 段落短，避免空话
- 中文回复，命令用代码块

## Boundaries
- 涉及外发消息、删除数据时必须确认
- 不编造事实，不伪造执行结果
- 不在群聊里代表用户说话
- 私人内容绝对保密
```

写 SOUL 的关键：

- **少写形容词，多写行为规则**："简洁"不如"每段不超过 3 行"
- **"不做什么"比"做什么"更重要**：明确列出禁止项
- **规则不要互相冲突**：避免"既要主动又不能打扰"

你可以直接让助手帮你填写：

```text
请根据我们的对话风格，帮我生成一份 SOUL.md
```

---

## 3. IDENTITY.md：定义自我认知

让助手对自己有清晰定位：

```markdown
# IDENTITY

- 名字：（取一个你喜欢的名字）
- 调性：（犀利？温暖？沉稳？）
- 签名 Emoji：（选一个代表它的 emoji）
```

建议在首次对话时就让助手自己填写。它会根据你们的互动风格，自己取名字、定调性。

---

## 4. USER.md：把你的信息写成可执行上下文

USER.md 不是简历，是让助手减少反问、直接进入执行的上下文。

推荐模板：

```markdown
# USER

## Profile
- Role: （你的角色）
- Timezone: Asia/Shanghai

## Current Focus
- Project A: （当前在做什么）
- Project B: （第二优先级）

## Preferences
- 输出先给结论
- 提供可执行命令
- 变更前先说明影响范围
```

你写得越具体，助手越少反问，越能直接进入执行状态。这个文件会随交互逐步丰富。

---

## 5. 重启并验证

改完三件套后重启：

```bash
openclaw daemon restart
```

用同一个问题做 A/B 对比，例如：

- "帮我写一封项目更新邮件"
- "给出风险点和上线检查清单"

你要看到的变化：

- 语气和结构更稳定
- 输出更符合你的偏好
- 安全边界触发更明确

![灵魂三件套关系图](/images/openclaw/02-soul-trinity.png)

---

## 6. 持续迭代

三件套不是一次写完就结束。每次出现"它应该这样做，但它没做到"的瞬间，就是更新的时机。

建议：

- 每次调整后跑一条固定回归问题
- 每周清理一次冲突规则
- 让助手在修改 SOUL.md 时主动告诉你改了什么

当三件套稳定后，助手才会从"会回答"升级为"会协作"。
